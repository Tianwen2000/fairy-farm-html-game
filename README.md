# 🌻 精灵农场 · Sprite Farm

> 一个**单 HTML 文件**的种田休闲小游戏：养精灵、种作物、升级解锁更多地块。
> **纯前端 + 内置 SVG 美术，零依赖**，双击即玩。

养成你的小精灵🧚，每只精灵只能种植与自己属性相同的作物🌾🌽🍅🎃。
卖作物赚金币💰，用金币升级精灵、解锁新伙伴，最多带 3 只上阵开垦农田。

---

## ✨ 特性

- 🧩 **零依赖单文件**：一个 `index.html`，无需构建、无需后端、无需联网，复制到任意浏览器即可运行
- 🎨 **内置 SVG 美术**：精灵 / 作物 / 土地全部用矢量 SVG 绘制，随窗口缩放清晰不糊
- 🐣 **多帧动画 & 特效**：精灵待机浮动、抚摸时切到「开心帧」Q 弹；种植 / 收获有粒子与上浮文字特效
- 🌱 **完整养成循环**：种植 → 生长 → 收获 → 升级 → 解锁 → 换场
- 🏆 **奖励地块机制**：上阵精灵不满 3 只时获得额外耕地；满队时智能回收并提示
- 📱 **移动端友好**：自适应布局，手机浏览器同样顺滑


## 🎮 玩法说明

### 核心循环

```
抚摸精灵拿种子  →  点击空地种植  →  等待生长  →  点击成熟作物收获金币
        ↑                                                      │
        └──────────  升级精灵 / 商城解锁新精灵  ←──────────────┘
```

### 1. 精灵系统 🧚
- 每只精灵有独立**等级 Lv.1 ~ Lv.6**，等级决定可耕种的**地块数量**（每升 1 级 +1 块地）
- 升级消耗金币，成本随等级递增：`100 → 250 → 500 → 800 → 1200`
- 升级同时让作物**生长时间缩短**（每级 -5%，上限 -30%）、**售价提升**（每级 +5%）

### 2. 种田系统 🌾
- 地块三态：**空闲 → 生长中 → 成熟**
- 点击空地消耗 1 个对应种子开始种植，生长采用**真实时间**倒计时
- 成熟后地块高亮跳动，点击收获；也可用「🧺 一键收获」一次性收完
- 收获金币 = `基础售价 × (1 + (等级 - 1) × 0.05)`

### 3. 种子系统 🌱
- 种子**只能通过「🤚 抚摸」精灵获得**，不可购买
- 每次抚摸随机 `1~3` 个；精灵 Lv≥2 额外 +1，Lv≥4 再 +1（单次上限 6）

### 4. 商城 🛒
- 出售新精灵的**解锁权**，购买后存入「精灵小屋」，需手动上阵
- 显示「已拥有 / 可解锁 / 金币不足」状态

### 5. 精灵小屋 🏠 & 换场
- 展示所有已解锁精灵，支持上阵 / 换下 / 替换
- **最多同时上阵 3 只**；换下的精灵作物会被**冻结保留**，上场后继续生长

### 6. 奖励地块 🎁（特色机制）
- 场上**恰好 2 只**精灵时，每只额外获得 **+1 块奖励地**
- **开局 / 单只在场**：严格按等级数地块（开局 1 块）
- **补满第 3 只**时奖励地收回：优先回收空地；若某精灵已无空地，会弹窗让你**选择清空哪一块作物**（可取消上场，作物零损失）

## 🛠️ 技术实现

- **纯 HTML / CSS / JavaScript**，无任何外部依赖或框架
- 游戏状态用普通 JS 对象维护；事件统一通过 `data-action` **委托绑定**
- 生长 / 倒计时由单个 `setInterval` 周期刷新驱动，换场时正确冻结 / 解冻计时
- 全部美术为**内联 SVG**，配合 CSS `@keyframes` 实现动画与特效

```
html纯前端游戏/
├── index.html      # 游戏本体（HTML + CSS + JS + SVG 全在这一个文件里）
└── README.md       # 本文档
```

## 🎨 自定义美术（换成你自己的图片）

美术集中在 `index.html` 的三个函数 + 一个配色表，均有注释：

| 函数 / 配置 | 作用 |
|------|------|
| `ART` | 各精灵的配色（body / 描边 / 腮红） |
| `spriteSVG(id, pose)` | 精灵头像，`pose` 为 `'idle'` / `'happy'`（多帧动画） |
| `cropSVG(id, stage)` | 作物，`stage` 为 `'sprout'` / `'growing'` / `'mature'` |
| `soilSVG(moist)` | 土地贴图（干 / 湿） |

想替换为自己的 PNG / GIF，把对应函数的返回值改成 `<img src="...">` 即可；
多帧动画沿用 `pose` / `stage` 参数的思路传入不同图即可。
